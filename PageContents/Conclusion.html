Conclusion


In the future, we plan on creating new ways to be able to link virtual connections to face-to-face connections. Currently, we have been building features like events and posts to be able to achieve this kind of link. Another future plan that we would like to possibly expand is to build unique ways of making each user have a sense of identity which can help our users be able to build deeper connections.

The very next step that we are taking after the final milestone would be flushing out the functionality that we currently have which includes things such as Admins/moderator permissions, different communities, and possibly a new explore map view. The biggest functionality that we would want to work further on is developing the interaction between different communities and people. As well as possibly making communities have a more specific identity. A way we possibly tackle this is by forming new communities based on location (I.E San Francisco, Bay Area, San Jose)

We learned that designers like ourselves should focus on both functionality and aesthetic choices. We want to make sure intended users can easily use applications to find new communities and connections. This is the functionality that we want to focus on, while we also want to focus on how the application looks. An example of this is how we made the application more readable with certain colors. If we continue to build our application, we will apply techniques that we learned from the 4 milestones.
